História dos computadores

A história dos computadores é muito antiga. Desde o ábaco e máquinas de calcular como a de Pascal, já existia o desejo de facilitar cálculos e registros. Com o tempo, surgiram os primeiros computadores mecânicos, depois os elétricos e, finalmente, os digitais.

Na década de 1940, o ENIAC marcou o início da era digital. Era enorme, mas já realizava cálculos automáticos. Hoje, os computadores estão por toda parte e são essenciais para a vida moderna.

Aprender a base histórica me ajudou a entender como a tecnologia evoluiu e o quanto a linguagem HTML é recente nesse processo.

O que são Clients?

Clients (clientes) são os dispositivos ou programas que fazem pedidos a servidores. Um exemplo de client é o seu navegador de internet, como o Chrome ou Firefox. Eles pedem ao servidor os arquivos de um site e exibem para o usuário.

Aprendi que o client nunca funciona sozinho; ele depende da resposta do servidor para funcionar corretamente. É como se o client fosse o "aluno" e o servidor o "professor" respondendo as dúvidas.

O que são Servers?

Servers (servidores) são computadores que armazenam e enviam dados para os clients. Eles estão sempre "ligados" e prontos para responder às requisições.

Um site, por exemplo, está hospedado em um servidor. Quando você acessa esse site, seu navegador (client) pede os dados e o servidor responde com o conteúdo.

Entender a relação entre client e server foi essencial para perceber como as páginas funcionam na prática.

Estrutura básica do HTML

HTML significa HyperText Markup Language. Ele é usado para estruturar o conteúdo de uma página web. Toda página começa com <!DOCTYPE html>, que define o tipo do documento.

Depois vem a tag <html>, que envolve tudo. Dentro dela, temos o <head> com configurações invisíveis, como título e charset, e o <body>, onde fica todo o conteúdo visível: textos, imagens, listas, links, etc.

Aprendi que a organização do HTML é como um esqueleto que sustenta todo o site.

Listas ordenadas e não ordenadas

As listas são usadas para organizar informações. A lista ordenada usa a tag <ol> e mostra os itens em ordem numérica ou alfabética:

  1. Primeiro item
  2. Segundo item

Já a lista não ordenada usa a tag <ul>, que coloca marcadores (bolinhas):

  • Item A
  • Item B

Com isso, aprendi a apresentar conteúdos com organização visual simples.

O que é Semântica no HTML?

Semântica no HTML significa dar sentido ao conteúdo da página usando as tags apropriadas. Por exemplo, em vez de usar apenas <div> para tudo, o HTML5 trouxe novas tags como:

  • <header> — representa o cabeçalho da página ou de uma seção.
  • <nav> — representa os links de navegação.
  • <main> — conteúdo principal do site.
  • <article> — conteúdo independente e completo.
  • <aside> — informações complementares (como uma barra lateral).
  • <footer> — rodapé da página.

Essas tags ajudam os navegadores, desenvolvedores e leitores de tela a entenderem a função de cada parte da página. Isso é essencial para acessibilidade e organização.

O que é Acessibilidade na Web?

Acessibilidade na web significa garantir que todas as pessoas possam acessar e entender o conteúdo, incluindo pessoas com deficiências visuais, motoras ou cognitivas.

Para isso, usamos boas práticas como:

  • Adicionar descrições com alt nas imagens:
    <img src="imagem.jpg" alt="Descrição da imagem">
  • Usar estrutura semântica com tags adequadas (como <main>, <article>, <nav>).
  • Evitar elementos visuais confusos ou que dependam apenas de cores.

Diferenças entre HTML4 e HTML5

O HTML4 era mais limitado e dependia muito da tag <div> para estruturar a página. Já o HTML5 trouxe novos elementos semânticos e suporte nativo para áudio, vídeo e acessibilidade.

Algumas tags antigas foram descontinuadas (deprecated), como:

  • <font>
  • <center>
  • <basefont>
  • <big>
  • <strike>

Essas tags foram substituídas por CSS e pelas novas tags estruturais do HTML5.

Exemplo de código moderno e semântico:

<main>
  <header><h1>Título do site</h1></header>
  <article>
    <h2>Conteúdo principal</h2>
    <p>Texto do artigo...</p>
  </article>
  <footer>Rodapé do site</footer>
</main>
